# ADR-006: License under Apache-2.0

**Status:** Accepted · **Date:** 2026-07-15

## Context

The project is committed to being open-source from day one ([vision.md](../vision.md)), which requires a LICENSE file before the first external contribution or fork can happen in good standing. The choice is between a permissive license (MIT, Apache-2.0) and a copyleft license (AGPL-3.0) that would force anyone hosting a modified version as a network service to release their source.

Given [ADR-001](ADR-001.md) — this is a client-side tool with no server component to "run as a service" in the first place — the AGPL network-use clause has little to bite on for v1: there's no server-hosted version of this tool for AGPL to force open. A permissive license better serves the stated goal of being easy for contributors and forks to build on without legal friction.

## Decision

License the project under **Apache License 2.0**. Chosen over MIT for the explicit patent grant (Section 3), which gives users and contributors some protection against patent claims related to contributed code — relevant given the project touches AI provider integrations, a space with active patent activity.

## Consequences

- **Positive:** low-friction for contributors and forks, explicit patent grant, compatible with essentially all dependency licenses this project is likely to use (React, Vite, provider SDKs).
- **Negative:** nothing prevents a third party from hosting a commercial, closed-source SaaS built on this code without contributing back. Accepted as consistent with "open-source friendly" outweighing that risk at this stage; revisit if the project grows a server component that changes the calculus (see [ADR-002](ADR-002.md) Follow-up).
- **Action:** `LICENSE` file added at repo root; `NOTICE` file to be added if/when third-party Apache-2.0-licensed code is vendored in.
