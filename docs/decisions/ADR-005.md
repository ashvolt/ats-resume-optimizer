# ADR-005: Migrate to TypeScript

**Status:** Accepted · **Date:** 2026-07-15

## Context

The v0.1 codebase is a single 1035-line untyped JSX file. The v1 architecture introduces several contracts that are load-bearing across module and contributor boundaries: the `ProviderAdapter` interface ([ADR-003](ADR-003.md)), the Resume JSON and JD JSON schemas ([architecture.md § Domain Model](../architecture.md#domain-model-and-data-schema)), the storage interface ([ADR-002](ADR-002.md)), and export-format plugins. These are exactly the kind of interfaces where an undetected shape mismatch (a provider adapter returning the wrong field name, a parser producing a Resume JSON that's missing a field the editor expects) causes silent runtime bugs rather than a compile error — and this project explicitly wants outside contributors implementing these interfaces without deep codebase familiarity.

## Decision

Migrate to TypeScript for all new code. Core schemas (Resume JSON, JD JSON, ProviderAdapter, StorageAdapter, ExportAdapter) are defined as TypeScript types/interfaces and are the canonical source of truth — `docs/` schema examples are generated from or kept in lockstep with these types, not hand-duplicated. Existing JS files are migrated incrementally as they're touched during the rearchitecture (no separate "convert everything" pass), starting with the modules in [architecture.md § Component Breakdown](../architecture.md#component-breakdown) that own the interfaces above.

## Consequences

- **Positive:** provider/parser/export plugins get compile-time contract checking; schemas are self-documenting; better contributor onboarding via editor autocomplete and inline types instead of reading implementation to infer shape.
- **Negative:** migration cost, slightly higher barrier for drive-by contributors unfamiliar with TypeScript (mitigated by keeping plugin interfaces small and well-documented in [features/](../features/)).
- **Scope:** this is a language/tooling decision, not a rewrite mandate — see [ROADMAP.md](../../ROADMAP.md) for sequencing relative to the rest of the v1 work.
