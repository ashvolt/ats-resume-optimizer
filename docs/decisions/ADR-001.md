# ADR-001: Standalone, backend-optional web app (leave the Claude Artifact runtime)

**Status:** Accepted · **Date:** 2026-07-15

## Context

The v0.1 prototype (`ats-resume-builder.jsx`) runs as a Claude.ai Artifact: Google Docs/Gmail export works via `window.sendPrompt(...)`, a function only present when the component is rendered inside a Claude chat session with MCP connectors enabled. A standalone Vite build exists (`src/main.jsx`) but stubs that call to a clipboard-copy fallback — it was a secondary target, not the primary one.

The new vision requires: BYOM to arbitrary AI providers (including local models via Ollama/LM Studio), secure credential storage, PDF upload/parsing, a plugin architecture for parsers/exporters, and an open-source project other people can run and contribute to. None of these are achievable inside an Artifact sandbox:

- Artifacts have no guarantee of arbitrary outbound `fetch` to third-party AI provider domains or to `localhost` (where Ollama/LM Studio serve).
- Artifacts have no durable, user-controlled storage suitable for API keys or version history beyond a small sandboxed key-value store.
- An Artifact only runs inside a Claude.ai chat session — it cannot be self-hosted, cannot be `git clone`'d and run by a contributor, and cannot be the basis of an independent OSS project with its own release cadence.

## Decision

The application becomes a standalone, independently deployable web app (static build, deployable to any static host or run locally via `vite dev`/`vite preview`), with **no mandatory backend**. It no longer targets the Claude Artifact runtime as a first-class environment.

"No mandatory backend" means: the app must fully function — import, score, optimize (if a provider is configured), edit, export — with nothing running server-side. A backend may exist later as an *optional* add-on (see [ADR-002](ADR-002.md)), never as a requirement to use the tool.

## Consequences

- **Positive:** unblocks BYOM/local-model support, unblocks PDF parsing without uploading resumes anywhere, matches the "privacy-first, no signup, no cost" principle in [vision.md](../vision.md), makes the project trivially self-hostable and forkable.
- **Negative:** loses the current Google Docs/Gmail "push via Claude" integration as-is — tracked separately in [ADR-007](ADR-007.md). Any future integration requiring a server-side secret (e.g., an OAuth client secret that can't live in a public static bundle) needs its own design, not a default assumption.
- **Follow-up:** `ats-resume-builder.jsx` and its `window.sendPrompt` dependency are deprecated (see [CHANGELOG.md](../../CHANGELOG.md)) in favor of the modular structure in [architecture.md](../architecture.md).
